很多论文同时碰多个点，但都有一个主矛盾。

一、优化 Drafter：怎么更快、更准地“猜”
工作	标签	核心优化	本质定位
Medusa / MTP	D、Q、T、R	在 target 上附加多个未来 token 预测头，一次预测多个位置，再组织成候选树供 Verify	不需要独立小模型，属于“多头并行预测”路线。Medusa 同时奠定了多头＋Tree Attention 的基本形态。
EAGLE-1	Q、R	不直接预测 token，而是在 target 的特征空间里自回归预测未来 feature	目标是比普通小模型更贴近 target，但 Draft 仍然串行
EAGLE-2	Q、T、V	在 EAGLE Feature Drafter 上加入动态 Draft Tree，根据置信度调整分支	把重点从“固定树”转向“每轮自适应树”
EAGLE-3	Q、R	改成直接预测 token；融合 target 的浅、中、深层特征；用 Training-Time Test 模拟推理时多步 rollout	当前自回归 Feature Drafter 的代表性强基线。
Beagle	Q、R、D	用 Cross-Attention Drafter 代替 EAGLE 式紧耦合 self-attention/fusion，并设计两阶段 block-attention 训练	主攻模型结构简化、训练稳定性和训练显存，不是推理机制上的巨大变化。
PARD	D、R、Q	把现成 AR 小模型低成本改造成一次预测多个位置的并行 Drafter；一个 Drafter 可服务同模型家族多个 target	主攻低成本适配和 target-independent，而非强绑定 target hidden state。
DiffuSpec	D、Q、V	直接复用预训练 diffusion LM 一次生成 token lattice；用因果一致路径搜索选出 AR-compatible 路径，并动态调 Draft Length	这里的 Drafter 最接近“真正的预训练扩散模型”。
SpecDiff-2	D、Q、R	使用离散扩散模型做非自回归 Drafter，并专门校准 Diffusion Drafter 与 AR Verifier 的分布对齐	重点是解决“扩散并行但和 target 不对齐”的问题。
DFlash	D、Q、R	轻量 Block-Diffusion Drafter 一次预测整块；抽取 target 多层 hidden state，经融合后通过 KV injection 注入每个 Draft 层	主攻把 Drafting 从多次串行 forward 压成一次并行 forward，同时用 target 内部知识补偿小模型容量。
DFLARE	Q、R、D	每个 Draft 层学习不同的 target-layer 融合权重；target context 与 Draft Token 使用异构 KV projection；增加 Draft 深度和训练数据；渐进式位置加权损失	不是改变 DFlash 流程，而是解除 DFlash 的“所有 Draft 层吃同一份 target feature”容量瓶颈。
Domino	D、Q、R	并行 backbone 先输出整块基础分布，再用轻量 causal head 根据已生成前缀顺序修正 logits；配套 base-anchored curriculum	解决并行 Drafter 缺少 block 内因果依赖的问题，核心是“昂贵计算并行、便宜修正串行”。
DSpark	D、Q、V、S、R	DFlash 式并行 backbone 后加 Markov/RNN 顺序头，缓解 suffix decay；同时训练 confidence head，并结合实时 SPS 曲线决定每个请求的 Verify 长度	是目前覆盖面最广的一类：既改 Drafter，也改 Verify 调度和高并发 serving。
这一组内部的真实关系
Medusa/MTP
  └─ 多头并行，但预测能力有限

EAGLE
  └─ 用 target feature 提高 Draft 质量，但 Drafting 仍串行

PARD / DART / DFlash
  └─ 把未来多个位置改为一次并行预测

DFLARE
  └─ 提高 DFlash 并行 backbone 容量

Domino / DSpark
  └─ 承认纯并行缺因果依赖，再补一个便宜的顺序模块

所以 DFLARE 解决“并行模型不够强”，而 Domino/DSpark 解决“并行模型结构上不知道前面实际选了什么”。这两个不是一回事，可以叠加。

二、优化候选结构：不只猜一条，而是决定猜哪些路径
工作	标签	核心优化	本质定位
SpecInfer	T、V、S	多个小模型的候选合并或扩展成 token tree；target 用 Tree Attention 一次完成整棵树的 Verify	奠定了“树状投机＋一次并行 Verify”的系统路线。
Medusa	T、Q	多个 prediction head 产生不同未来位置候选，再预构造树供 Verify	候选来自多头，不是完整 Drafter Rollout。
EAGLE-2	T、Q	根据 Draft 概率动态扩树，避免固定树对所有样本一刀切	自回归 Drafter 下的动态树代表
OPT-Tree	T、V	在固定节点预算下，搜索最大化期望接受长度的树结构	后续 DDTree 等“概率排序＋节点预算”方法的重要基础。
DART	D、Q、T	从 target hidden state 一次预测多个未来位置的 logits，再用 N-gram 连贯性约束剪成候选树	同时改 Drafter 和树；外部 N-gram trie 帮它补并行位置之间的语义连续性。
DDTree	T、V	直接利用 DFlash 的逐位置边缘分布，用 best-first heap 在固定 node budget 下选概率质量最高的前缀节点，再用 Tree Attention 执行 Verify	将 DFlash 从“单路径”扩展到“多路径”，但仍假设各位置分布可因式分解。
TAPS	T、V、Q	不再只按逐位置 marginal probability 选节点，而是估计 prefix-conditioned acceptance，减少前缀已经失败却仍被加入的后代节点	针对 DDTree 的核心缺陷：高边缘概率不等于真实路径容易存活。
SMART	T、V、S、H	不以接受长度最大为目标，而是比较每次扩树的边际收益和硬件 Verify 成本，只在值得时扩节点	从“概率最优树”转向“端到端速度最优树”，尤其关注 batch 和硬件饱和。
Progressive Tree Drafting	D、T、V	不附加独立 Drafter，而是引导 target 本身在一次 forward 中逐步构造并剪枝多条路径	training-free、model-agnostic，属于 target 自投机和结构化并行。由于是 2026 年 7 月刚发布，结论还需要更多复现。
树方向其实分三层
第一层：树从哪里来
SpecInfer：多个小模型/候选序列合并；
Medusa：多个预测头；
EAGLE-2、OPT-Tree：自回归 Drafter 展开；
DART、DDTree：一次并行 logits 展开。
第二层：树里选哪些节点
OPT-Tree：最大化期望接受长度；
DDTree：按 diffusion marginal 的前缀概率选；
TAPS：加入真实 prefix survival；
SMART：加入硬件成本。
第三层：树该多大
DDTree 通常先给固定预算；
SMART 直接根据边际收益/成本动态停；
DSpark 虽不是树方法，但做了类似的“Verify 多长才划算”。
三、优化 Verify：不是猜得越多就要做越多 Verify
工作	标签	核心优化	解决的问题
DiffuSpec ADL	V	根据在线接受反馈调整 Proposal Length	输入难度不同，固定 Draft Length 会浪费
TAPS	T、V	选择真正有 Prefix Survival 价值的节点	避免把不可达或低收益分支送入 Verify
SMART	V、T、H	根据 target 执行 Verify 的真实边际成本控制树大小	避免接受长度涨了，但 wall-clock 反而变慢
DSpark	V、S	confidence head 预测逐位置条件接受率；结合当前 batch 的 SPS 曲线，为不同请求分配不同 Verify Length	高并发时，Verify Token 会占用昂贵的 batch capacity；并非“多做 Verify 基本免费”。
DFlash / DFLARE	—	默认固定 block 单路径 Verify	主要优化 Draft，不直接解决 Verify Budget

这里最关键的区别是：

DDTree/TAPS/SMART：决定一棵树里放哪些节点；
DSpark：决定每个请求的链式 Prefix 的 Verify 长度；
Saguaro：不减少 Verify，而是把 Verify 等待时间藏起来。
